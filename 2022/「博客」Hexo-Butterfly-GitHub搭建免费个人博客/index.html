「博客」Hexo+Butterfly+GitHub搭建免费个人博客

发表于2022-07-02|更新于2022-07-03|博客
|字数总计:693|阅读时长:2分钟|阅读量:

打勾的为必备选项，没打勾的为可选选项

建议全程挂代理

软件

  • 「Node」，主要用到「Node.Js」的包管理器「Npm」

    [Node.js (nodejs.org)]

  • 「Git」，所有的命令都在「Git Bash」中进行

    [Git (git-scm.com)]

  • 「VsCode」，用于编辑文本，在安装「Git」后也可以输入「Git Bash」命令

仓库

创建本地仓库

「Hexo框架」

[Hexo]

  1. 新建本地博客目录「博客」

  2. 右键「博客」➪「Git Bash Here」

  3. 「Npm」安装「Hexo」

  4. 初始化「Hexo」

  5. 启动本地服务器预览

「Butterfly主题」

[Butterfly - A Simple and Card UI Design theme for Hexo]

  1. 安装主题

    • 「Github」安装

    • 「Gitee」安装

    • 「Npm」安装

  2. 找到博客\themes\butterfly下的「_config.yml」,复制并重命名为「_config.butterfly.yml」，粘贴到「博客」下

  3. 「_config.yml」修改，启用主题

  4. 删除自带的「Landscape」主题

    • 删除「_config.Landscape.yml」

    • 删除「node_modules」下的「hexo-theme-landscape」文件夹

  5. 「Butterfly」必须安装插件「Pug渲染器」

  6. 启动本地服务器预览

创建远程仓库

  1. 注册并登录「GitHub」
  2. 新建一个与用户名同名的空仓库：「用户名.gitbub.io」
  3. 点击仓库「用户名.gitbub.io」➪「Settings」➪「Pages」，勾选「Enforce HTTPS」

绑定

  1. 回到「Git Bash」，设置「用户名」和「邮箱」

  2. 创建「SSH」密钥，用于本地计算机和远程的连接

    连按三个「回车」，设置为空密码

  3. 进入C:\Users\用户名\.ssh目录，「.ssh」文件夹默认隐藏，打开它，找到「 id_rsa.pub」,复制里面的内容

  4. 登录「GitHub」➪「Setting」➪「SSH and GPG keys」➪点击「New SSH key」➪填入复制的内容并保存

  5. 回到「Git Bash」验证连接，至此本地电脑与「GitHub」绑定成功

  6. 找到「博客」下的「_config.yml」，拖到最下面修改，至此本地仓库与「GitHub」绑定成功

  7. 安装「Hexo Deployer」布署插件

    布署，本地网站上传到互联网

域名

  • 「GitHub」提供的免费域名「用户名.github.io」

  • 花十几元就能买到一年的付费域名，如腾讯云

    将买到的域名如「Minge.live」填入一个文本文件，删除文件后缀，并重命名为「CNAME」放到博客/source下，布署完成后，打开「用户名.github.io」会自动跳转到「minge.live」，当然也可以直接打开「https://minge.live」
